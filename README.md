# Sarvpratham Backend — Deployment Guide (Render)

This is a Python (FastAPI) backend with a PostgreSQL database. It handles:
- Enquiry form submissions (`/api/enquiries`)
- Slot booking with server-side double-booking prevention (`/api/bookings`)
- Admin endpoints to view/manage data (`/api/enquiries`, `/api/bookings`, `/api/admin/stats`, CSV export) — protected by login

## 1. Push this `backend` folder to GitHub

Render deploys from a GitHub repo. Create a new repo (e.g. `sarvpratham-backend`) and push just the contents of this `backend` folder to it.

## 2. Create the database on Render

1. Go to https://dashboard.render.com → New → PostgreSQL
2. Name it `sarvpratham-db`, choose the Free plan, and create it
3. Once created, copy the "Internal Database URL" — you'll need it in step 3

## 3. Create the web service on Render

1. New → Web Service → connect your GitHub repo
2. Runtime: Python 3
3. Build command: `pip install -r requirements.txt`
4. Start command: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`
5. Add environment variables (Settings → Environment):
   - `DATABASE_URL` = the Internal Database URL from step 2
   - `SECRET_KEY` = any long random string (Render can generate one for you)
   - `ADMIN_USERNAME` = pick a username
   - `ADMIN_PASSWORD` = pick a strong password
   - `ALLOWED_ORIGINS` = `https://sarvprathameduconsultants.com,https://www.sarvprathameduconsultants.com`
6. Click Create Web Service

Render will build and deploy. You'll get a URL like `https://sarvpratham-backend.onrender.com`.

Alternatively, if your repo includes `render.yaml` (already included here), you can use "New → Blueprint" on Render and it will set up both the database and web service automatically — you'll just need to fill in `ADMIN_USERNAME` and `ADMIN_PASSWORD` when prompted.

## 4. Point your frontend at the backend

In `index.html`, find this line near the bottom (just before `main.js` loads):

```html
window.SP_API_BASE = 'https://sarvpratham-backend.onrender.com';
```

Replace it with your actual Render URL from step 3.

## 5. Test it

Visit `https://YOUR-BACKEND-URL/api/health` — you should see `{"status":"ok"}`.

Then test the API docs (auto-generated by FastAPI) at:
`https://YOUR-BACKEND-URL/docs`

From there you can try `/api/admin/login` with your admin username/password to get a token, then use that token (via the "Authorize" button in the docs page) to try `/api/enquiries`, `/api/bookings`, and `/api/admin/stats`.

## 6. Upload the frontend

Upload `index.html`, `main.js`, and `styles.css` to your existing website hosting, replacing the current files.

## Notes

- **Free tier sleep**: Render's free web services spin down after inactivity and take ~30-60 seconds to wake up on the next request. The first enquiry/booking after a quiet period may feel slow — this is normal on the free tier. Upgrading to a paid plan removes this.
- **Database tables**: Tables are created automatically the first time the backend starts (no manual migration step needed for this size of project).
- **No admin dashboard UI yet**: The original site's admin dashboard panel (table of enquiries/bookings, stats, search, CSV button) wasn't present in the uploaded HTML, so it's not included here. The backend still has all the admin endpoints (list, update status, delete, stats, CSV export) — you can use them via `/docs`, curl, Postman, or by viewing the database directly (e.g. Render's PostgreSQL dashboard, or a client like TablePlus/DBeaver/pgAdmin connected with the External Database URL from your Render Postgres page). If you'd like an actual dashboard page built later, just ask.
- **Changing admin password**: just update the `ADMIN_PASSWORD` environment variable on Render and redeploy/restart — no code change needed.
